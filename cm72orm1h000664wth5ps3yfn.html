Maven (二) Maven 安装与配置

Maven 2025-02-10 15 分钟阅读 0

第二章： Maven 安装与配置

2.1 检查 JDK 安装

Maven 是基于 Java 开发的，所以首先要确保你的电脑上已经安装了 Java Development Kit (JDK)。 Maven 3.3.0 及以上版本需要 JDK 1.7 或更高版本。 推荐使用 JDK 1.8 或 JDK 11。

检查 JDK 版本步骤：

  1. 打开终端或命令提示符。

  2. 输入命令： java -version

    你应该看到类似下面的输出 (版本号可能不同):

    • 如果你看到了 Java 版本信息，说明 JDK 已经安装。 请确保版本号是 1.7 或更高。
    • 如果你看到错误信息，例如 “java 不是内部或外部命令，也不是可运行的程序”，说明 JDK 没有安装或者没有配置到环境变量中。 你需要先安装 JDK 并配置好环境变量。 (JDK 安装和环境变量配置超出了本 Maven 教程的范围，你可以搜索相关的教程来完成 JDK 的安装和配置)

确保 JDK_HOME 环境变量 (可选但推荐):

除了 PATH 环境变量，设置 JAVA_HOME 或 JDK_HOME 环境变量也是一个好的习惯，尤其是在需要配置多个 Java 版本时。 Maven 和其他 Java 工具可能会使用 JAVA_HOME 或 JDK_HOME 来找到 JDK 的安装目录。

设置 JAVA_HOME 或 JDK_HOME 环境变量 (以 Windows 为例):

  1. 找到你的 JDK 安装目录。 例如 C:\Program Files\Java\jdk1.8.0_xxx
  2. 打开 “控制面板” -> “系统与安全” -> “系统” -> “高级系统设置” -> “环境变量”。
  3. 在 “系统变量” 中，点击 “新建”。
    • 变量名：JAVA_HOME (或者 JDK_HOME)
    • 变量值：你的 JDK 安装目录 (例如 C:\Program Files\Java\jdk1.8.0_xxx)
  4. 点击 “确定” 保存所有更改。

2.2 下载 Maven

  1. 访问 Apache Maven 官网下载页面: https://maven.apache.org/download.cgi
  2. 在 “Files” 部分，找到 “Binary zip archive” (例如 apache-maven-3.x.x-bin.zip)。
  3. 点击链接下载 ZIP 文件。 不要下载 “source code” 版本。

选择 Maven 版本：

  • 稳定版本 (Recommended): 通常选择官网推荐的稳定版本。
  • 最新版本 (Latest): 如果你想尝试最新的功能，可以选择最新版本，但可能存在一些未知的 bug。
  • 旧版本 (Older Versions): 如果你的项目需要兼容旧版本的 Maven，可以选择旧版本。 但通常不推荐使用过旧的版本。

下载完成后，你会得到一个 ZIP 压缩包，例如 apache-maven-3.x.x-bin.zip。

2.3 安装 Maven

Maven 的安装非常简单，只需要将下载的 ZIP 压缩包解压到你电脑的某个目录即可。 这个目录就是 Maven 的安装目录。

安装步骤：

  1. 选择一个目录作为 Maven 的安装目录。 例如：
    • Windows: C:\apache-maven-3.x.x (或者 D:\tools\apache-maven-3.x.x 等， 避免使用包含空格的路径，例如 “Program Files”)
    • macOS/Linux: /opt/apache-maven-3.x.x (或者 /usr/local/maven 等)
  2. 将下载的 ZIP 文件 apache-maven-3.x.x-bin.zip 解压到你选择的目录。

解压完成后，你的 Maven 就已经 “安装” 完成了。 实际上 Maven 是一个绿色软件，解压即可用，不需要额外的安装程序。

2.4 配置 Maven 环境变量

为了能够在命令行中直接运行 mvn 命令，我们需要配置 Maven 的环境变量。 我们需要设置 MAVEN_HOME 环境变量，并将 Maven 的 bin 目录添加到 PATH 环境变量中。

配置环境变量步骤 (以 Windows 为例，macOS/Linux 类似，参考之前的步骤):

  1. 打开 “控制面板” -> “系统与安全” -> “系统” -> “高级系统设置” -> “环境变量”。
  2. 在 “系统变量” 中，点击 “新建”。
    • 变量名：MAVEN_HOME
    • 变量值：你的 Maven 解压目录 (例如 C:\apache-maven-3.x.x)
  3. 在 “系统变量” 中，找到 “Path” 变量，双击编辑。
  4. 点击 “新建”，添加 %MAVEN_HOME%\bin
  5. 点击 “确定” 保存所有更改。

2.5 验证 Maven 安装

配置完环境变量后，我们需要验证 Maven 是否安装成功。

验证步骤：

  1. 打开新的终端或命令提示符窗口 (之前打开的窗口可能环境变量还没有生效，需要重新打开)。

  2. 输入命令： mvn -v 或 mvn --version

    如果安装成功，你应该看到类似下面的输出 (版本号可能不同):

    • 如果看到 Maven 版本信息，恭喜你，Maven 安装成功了！
    • 如果看到错误信息，例如 “mvn 不是内部或外部命令，也不是可运行的程序”，说明环境变量配置有问题。 请仔细检查环境变量配置步骤，确保 MAVEN_HOME 和 PATH 配置正确，并重新打开终端窗口。 如果仍然有问题，可以重启电脑试试。

2.6 Maven 本地仓库

Maven 本地仓库是 Maven 用来 缓存依赖 的地方。 当你第一次运行 Maven 命令需要下载某个依赖时，Maven 会从远程仓库下载依赖，然后 保存到本地仓库。 下次再需要这个依赖时，Maven 会直接从本地仓库读取，而不需要再次从远程仓库下载，这样可以 加速构建过程。

默认本地仓库位置:

  • 用户目录/.m2/repository (例如 Windows: C:\Users\YourUsername\.m2\repository, macOS/Linux: /Users/YourUsername/.m2/repository)

修改本地仓库位置 (通过 settings.xml 文件):

你可以通过修改 Maven 的 settings.xml 文件来更改本地仓库的默认位置。

  1. 找到 settings.xml 文件。

    • 全局 settings.xml: 位于 Maven 安装目录的 conf 目录下 (例如 C:\apache-maven-3.x.x\conf\settings.xml)

    • 用户 settings.xml: 位于 用户目录/.m2 目录下 (例如 C:\Users\YourUsername\.m2\settings.xml)

    • 优先使用用户 settings.xml: 如果用户目录 .m2 下没有 settings.xml 文件，你可以从全局 settings.xml 文件复制一份到用户目录 .m2 下，然后进行修改。 推荐修改用户 settings.xml，不要直接修改全局 settings.xml。

  2. 打开 settings.xml 文件，找到 <localRepository> 标签 (默认是被注释掉的)。

  3. 取消注释 <localRepository> 标签，并修改 <localRepository> 的值为你想要设置的本地仓库路径。

  4. 保存 settings.xml 文件。

    注意：修改本地仓库位置后，之前已经下载到旧本地仓库的依赖不会自动迁移到新的本地仓库。 你需要手动清理旧本地仓库，或者让 Maven 重新下载依赖到新的本地仓库。

2.7 Maven Settings 文件 (settings.xml)

settings.xml 文件是 Maven 的 全局配置文件，用于配置 Maven 的全局行为。 除了修改本地仓库位置，settings.xml 还可以配置：

  • 镜像 (Mirrors): 用于加速依赖下载，例如配置国内的 Maven 镜像。
  • 代理 (Proxy): 如果你的网络需要通过代理才能访问外部网络，可以在 settings.xml 中配置代理。
  • 服务器认证 (Servers): 配置访问私服仓库需要的用户名和密码。
  • Profile (全局 Profile): 可以在 settings.xml 中定义全局 Profile，并设置默认激活的 Profile。

我们会在后续章节中逐步学习 settings.xml 的更多配置。
