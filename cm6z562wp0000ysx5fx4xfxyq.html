Kotlin 基础语法 (Basics)

本章我们将学习 Kotlin 的基本语法元素，这是编写任何 Kotlin 程序的基础。

变量 (Variables): val 和 var

在 Kotlin 中，我们使用关键字 val 和 var 来声明变量，它们之间有一个重要的区别：

  • val (Value): 声明不可变变量 (Immutable Variable)

    • val 声明的变量一旦赋值后，就不能再被重新赋值。
    • 类似于 Java 中的 final 变量。
    • 推荐尽可能使用 val，因为它有助于编写更安全、更易于理解的代码。

  • var (Variable): 声明可变变量 (Mutable Variable)

    • var 声明的变量在赋值后，可以被重新赋值。
    • 类似于 Java 中普通的变量。

语法:

  • 变量名: 变量的名称，遵循标识符命名规则 (字母、数字、下划线，不能以数字开头)。
  • 数据类型: 变量存储的数据类型，例如 Int (整数), Double (浮点数), String (字符串) 等。 Kotlin 有类型推断，通常可以省略数据类型声明，编译器会自动推断。
  • 初始值: 变量的初始值。 val 变量必须在声明时赋值，var 变量可以先声明后赋值。

数据类型 (Data Types)

Kotlin 像 Java 一样，是一种静态类型语言，这意味着每个变量在编译时都必须有一个明确的数据类型。 Kotlin 提供了以下基本数据类型：

  • 数字类型 (Numbers):

    • 整数类型 (Integer Types):
      • Byte: 8 位有符号整数
      • Short: 16 位有符号整数
      • Int: 32 位有符号整数 (常用)
      • Long: 64 位有符号整数 (常用，用于表示较大的整数)
    • 浮点数类型 (Floating-Point Types):
      • Float: 32 位浮点数
      • Double: 64 位浮点数 (常用，精度更高)

  • 布尔类型 (Boolean):

    • Boolean: 表示真 (true) 或假 (false)。

  • 字符类型 (Character):

    • Char: 表示单个字符，用单引号 ' ' 包裹。

  • 字符串类型 (String):

    • String: 表示字符序列，用双引号 "" 包裹。

类型推断 (Type Inference): Kotlin 强大的类型推断能力允许你在很多情况下省略数据类型声明，编译器会根据初始值自动推断变量的类型。

显式声明类型 vs. 类型推断:

  • 显式声明类型: val name: String = "Alice" - 代码更清晰，尤其是在复杂情况下，可以提高代码可读性。
  • 类型推断: val name = "Alice" - 代码更简洁，适用于简单的情况。

通常，对于简单的情况，使用类型推断可以使代码更简洁。 对于复杂的情况或者需要明确类型意图时，显式声明类型会更好。

运算符 (Operators)

Kotlin 支持常见的运算符，与 Java 类似：

  • 算术运算符 (Arithmetic Operators):

    • + (加)
    • - (减)
    • * (乘)
    • / (除)
    • % (取模/余数)
    • ++ (自增)
    • -- (自减)

  • 比较运算符 (Comparison Operators):

    • == (等于)
    • != (不等于)
    • > (大于)
    • < (小于)
    • >= (大于等于)
    • <= (小于等于)

  • 逻辑运算符 (Logical Operators):

    • && (逻辑与 - AND)
    • || (逻辑或 - OR)
    • ! (逻辑非 - NOT)

  • 赋值运算符 (Assignment Operators):

    • = (赋值)
    • += (加后赋值)
    • -= (减后赋值)
    • *= (乘后赋值)
    • /= (除后赋值)
    • %= (取模后赋值)

注释 (Comments)

注释用于在代码中添加解释和说明，不会被编译器执行。 Kotlin 支持两种注释：

  • 单行注释 (Single-line comments): 以 // 开头，注释内容直到行末。
  • 多行注释 (Multi-line comments / Block comments): 以 /* 开始，以 */ 结束，可以跨越多行。

好的注释习惯:

  • 注释应该清晰、简洁、准确。
  • 解释代码的目的、逻辑、重要部分。
  • 不要注释代码的显而易见的功能 (例如 i++ // i 加 1 这样的注释就没必要)。
  • 在复杂逻辑、算法、或者容易混淆的地方添加注释。

字符串模板允许你在字符串中嵌入变量或表达式的值，使字符串拼接更加方便和易读。
