第四章： Maven 构建生命周期

4.1 什么是 Maven 构建生命周期？

Maven 构建生命周期 (Lifecycle) 是指 Maven 为了 构建项目 而定义的一套 标准的、预定义的构建流程。 它将项目的构建过程划分为一系列 相互独立的阶段 (Phase)。 每个阶段负责处理构建过程的某个特定方面。

Maven 生命周期解决的问题:

  • 构建过程标准化: Maven 生命周期定义了统一的构建流程，无论项目类型如何，构建过程都遵循相同的生命周期阶段。
  • 构建任务自动化: 开发者只需要执行一个 Maven 命令，Maven 就会自动按照生命周期顺序执行到指定的阶段，并完成之前的所有阶段，实现构建任务的自动化。
  • 构建过程可预测: 由于构建过程是标准化的，所以构建结果是可预测的，避免了手动构建过程中的不确定性。

4.2 Maven 的三大生命周期

Maven 定义了三个内置的生命周期 (Lifecycle):

  1. clean 生命周期: 用于 清理项目。 它的目的是清理之前构建生成的文件，例如 target 目录及其内容。
  2. default 生命周期: 用于 构建项目。 它包含了项目构建的核心阶段，例如编译、测试、打包、安装、部署等。 这是最常用的生命周期。
  3. site 生命周期: 用于 生成项目站点文档。 它包含了生成项目站点文档相关的阶段，例如生成项目报告、发布项目站点等。

注意： 这三个生命周期是 相互独立的。 执行 clean 生命周期不会执行 default 生命周期，反之亦然。 但是，default 生命周期内部的阶段是 有顺序的，执行后面的阶段会自动执行前面的所有阶段。

4.3 clean 生命周期

clean 生命周期只有一个阶段：

  • clean: 清理项目的输出目录 (默认是 target 目录)。

常用命令：

  • mvn clean: 执行 clean 生命周期，清理项目。

clean 生命周期的作用：

  • 清理构建环境: 在开始新的构建之前，先执行 mvn clean 可以确保构建环境的干净，避免旧的构建输出文件影响新的构建结果。
  • 解决构建问题: 有时候构建过程中出现一些奇怪的问题，例如编译错误、资源文件丢失等，执行 mvn clean 后重新构建可能可以解决这些问题。

4.4 default 生命周期 (核心生命周期)

default 生命周期是 Maven 最核心的生命周期，用于构建和部署项目。 它包含了多个阶段，这些阶段按照顺序依次执行。 下面列出 default 生命周期中常用的阶段 (完整阶段列表请参考之前的表格)：

  1. validate: 验证项目是否正确配置，检查 POM 文件和项目环境是否正确。
  2. compile: 编译项目的源代码。
  3. test: 运行单元测试。
  4. package: 将编译后的代码打包成可发布的格式 (例如 JAR, WAR, EAR)。
  5. verify: 运行集成测试，检查 package 是否有效且达到质量标准。
  6. install: 将 package 安装到本地仓库，供其他 Maven 项目使用。
  7. deploy: 将最终的 package 发布到远程仓库，供团队共享或部署到生产环境。

常用命令：

  • mvn validate: 执行 validate 阶段。
  • mvn compile: 执行 compile 阶段，以及 validate 阶段。
  • mvn test: 执行 test 阶段，以及 validate, compile 阶段。
  • mvn package: 执行 package 阶段，以及 validate, compile, test 阶段。
  • mvn install: 执行 install 阶段，以及 validate, compile, test, package, verify 阶段。
  • mvn deploy: 执行 deploy 阶段，以及 validate, compile, test, package, verify, install 阶段。

生命周期阶段的执行顺序：

当你执行 mvn <phase> 命令时，Maven 会按照 default 生命周期的顺序，从 validate 阶段开始，依次执行到指定的 <phase> 阶段，以及之前的所有阶段。

例如，执行 mvn package 命令，Maven 会自动按照顺序执行以下阶段：

validate -> initialize -> generate-sources -> process-sources -> generate-resources -> process-resources -> compile -> process-classes -> generate-test-sources -> process-test-sources -> generate-test-resources -> process-test-resources -> test-compile -> process-test-classes -> test -> prepare-package -> package

4.5 site 生命周期

site 生命周期用于生成项目的 站点文档。 Maven Site 插件可以根据项目的 POM 文件、源代码注释、项目报告等信息生成 HTML 格式的项目站点文档，方便项目文档的发布和共享。

site 生命周期常用的阶段：

  • site: 生成项目站点文档。 默认会在 target/site 目录下生成站点文档。
  • site-deploy: 将生成的站点文档部署到远程服务器 (例如 Web 服务器、FTP 服务器)。

常用命令：

  • mvn site: 执行 site 生命周期，生成项目站点文档。
  • mvn site-deploy: 执行 site-deploy 生命周期，部署项目站点文档。

site 生命周期的作用：

  • 生成项目文档: 自动化生成项目文档，减少手动编写文档的工作量。
  • 统一文档风格: Maven Site 插件提供了标准的文档模板，可以保证项目文档风格的统一性。
  • 方便文档发布: site-deploy 阶段可以将文档发布到远程服务器，方便团队成员和用户访问。

4.6 插件与目标 (Plugin and Goal)

Maven 的核心功能都是通过 插件 (Plugin) 来实现的。 每个生命周期阶段的具体任务都是由 插件目标 (Plugin Goal) 来执行的。

  • 插件 (Plugin): Maven 插件是 Maven 功能的扩展单元，它是一组预定义的代码，用于执行特定的构建任务。 例如，maven-compiler-plugin 插件用于编译源代码，maven-surefire-plugin 插件用于运行单元测试，maven-jar-plugin 插件用于打包 JAR 文件。

  • 目标 (Goal): 插件提供的具体功能单元。 每个插件可以包含一个或多个目标。 目标是插件可以执行的具体任务。 例如，maven-compiler-plugin 插件的 compile 目标用于编译源代码，testCompile 目标用于编译测试代码。 maven-jar-plugin 插件的 jar 目标用于打包 JAR 文件。

生命周期阶段与插件目标的绑定 (Plugin Binding):

Maven 将生命周期阶段与插件目标 绑定 (Binding) 起来，当 Maven 执行到一个生命周期阶段时，会自动执行与之绑定的插件目标。 Maven 已经为一些常用的生命周期阶段 默认绑定 了插件目标。 例如：

  • compile 阶段默认绑定了 maven-compiler-plugin:compile 目标。
  • test 阶段默认绑定了 maven-surefire-plugin:test 目标。
  • package 阶段默认绑定了 maven-jar-plugin:jar 目标 (对于 jar Packaging 项目)。

自定义插件绑定:

你可以在 pom.xml 文件中 自定义插件绑定，例如：

  • 修改默认绑定的插件目标: 例如，你可以修改 compile 阶段默认绑定的编译器插件，使用不同的编译器。
  • 绑定新的插件目标到生命周期阶段: 例如，你可以将代码检查插件的目标绑定到 validate 阶段，在项目验证阶段进行代码检查。
  • 在一个生命周期阶段绑定多个插件目标: 例如，你可以在 package 阶段同时执行多个打包插件，生成不同格式的 package 文件。

插件配置:

你可以在 pom.xml 文件中 配置插件，例如指定插件版本、插件参数等。 插件的配置会影响插件目标的执行行为。

总结第四章:

第四章我们学习了 Maven 的构建生命周期，包括 clean, default, site 三大生命周期，以及 default 生命周期中常用的阶段。 我们理解了生命周期阶段的执行顺序，以及插件和目标的概念，以及生命周期阶段与插件目标的绑定关系。 理解 Maven 生命周期是掌握 Maven 自动化构建的核心。
