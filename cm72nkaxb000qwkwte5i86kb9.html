Kotlin (五) | 继承

KOTLIN 2025-02-10 15 分钟阅读 0

继承 (Inheritance)

继承是面向对象编程 (OOP) 的另一个核心概念。 它允许我们创建一个新的类 (子类/派生类)，继承已存在的类 (父类/超类) 的属性和方法。 继承实现了代码重用，并建立了类之间的 “is-a” 关系。 例如， “狗 是 动物”，那么 Dog 类可以继承 Animal 类。

类的继承

在 Kotlin 中，默认情况下，类和类的成员 (属性和方法) 都是 final 的，这意味着它们不能被继承或重写。 要允许一个类被继承，或者允许类中的成员被子类重写，需要使用 open 关键字。

  • open 关键字: 用于修饰类和成员 (属性和方法)，表示它们是开放的，可以被继承和重写。

open 关键字

  • open 类: 使用 open 关键字修饰的类才能被其他类继承。
  • open 成员 (属性和方法): 使用 open 关键字修饰的属性和方法才能在子类中被重写 (override)。

方法重写 (Method Overriding)

子类可以重写 (override) 父类中 open 修饰的方法，以提供不同的实现。 在子类中重写方法时，需要使用 override 关键字。

  • override 关键字: 用于在子类中声明重写父类的方法或属性。 Kotlin 要求显式使用 override 关键字，以提高代码可读性和安全性，防止意外重写。

例子见 6.3 中的 Dog 和 Cat 类，它们都重写了 Animal 类的 makeSound() 方法。

super 关键字

在子类中，可以使用 super 关键字来调用父类的成员 (属性和方法) 或构造函数。

  • super.属性名: 访问父类的属性。
  • super.方法名(): 调用父类的方法。
  • super(): 在子类的构造函数中调用父类的构造函数。 如果子类有主构造函数，并且父类也有主构造函数，则子类的主构造函数必须直接或间接地调用父类的主构造函数。 这在继承类时已经体现了，例如 class Dog(override var name: String) : Animal(name) 中的 Animal(name) 就是调用父类 Animal 的主构造函数。

抽象类 (Abstract Classes)

抽象类是一种不能被实例化的类，只能作为其他类的父类使用。 抽象类可以包含抽象成员 (抽象属性和抽象方法) 和非抽象成员。

  • abstract 关键字: 用于修饰抽象类和抽象成员。

  • 抽象类: 使用 abstract 关键字修饰的类。 抽象类不能创建对象。

  • 抽象成员 (属性和方法): 使用 abstract 关键字修饰的属性和方法。 抽象成员没有实现 (没有方法体)，子类必须重写抽象父类的所有抽象成员才能成为具体类 (可以被实例化)。
