Kotlin (九) | 扩展函数和扩展属性

KOTLIN 2025-02-10 15 分钟阅读 0

扩展函数和扩展属性 (Extension Functions and Extension Properties)

扩展函数和扩展属性是 Kotlin 中非常强大且独特的特性。 它们允许你向已有的类添加新的函数和属性，而无需修改这些类的源代码，也无需使用继承。 这在很多情况下非常有用，例如当你需要向第三方库中的类添加一些自定义的功能时，或者当你想要为系统类 (例如 String, Int 等) 添加一些便捷的方法时。

扩展函数 (Extension Functions)

扩展函数允许你向已有的类添加新的函数，就像这些函数是类本身定义的一样。 实际上，扩展函数是在类的外部定义的，但可以通过特殊的语法在类的实例上像成员函数一样调用。

语法:

  • fun: 关键字，表示这是一个函数定义。
  • 接收者类型 (Receiver Type): 要扩展的类的类型，例如 String, Int, List<T>, 自定义类名等。 接收者类型放在 . 前面。
  • 扩展函数名: 扩展函数的名称，遵循标识符命名规则。
  • 参数列表: 扩展函数的参数列表，与普通函数参数列表相同。
  • 返回值类型: 扩展函数的返回值类型，与普通函数返回值类型相同。
  • 函数体: 扩展函数的函数体代码块。 在函数体中，可以使用 this 关键字访问接收者类型的实例 (即调用扩展函数的对象)。

扩展函数可以扩展任何类: 包括 Kotlin 标准库中的类 (例如 String, List, Map 等)，Java 类，以及自定义类。

扩展属性 (Extension Properties)

扩展属性与扩展函数类似，允许你向已有的类添加新的属性。 但扩展属性实际上并没有在类中添加新的存储空间 (backing field)。 扩展属性只是为已有的类提供了一种访问属性的语法，你需要为扩展属性提供 getter (至少要有 getter)，如果是 var 类型的扩展属性，还需要提供 setter。

语法:

  • val/var: 声明扩展属性为只读 (val) 或可变 (var)。
  • 接收者类型: 要扩展的类的类型。
  • 扩展属性名: 扩展属性的名称，遵循标识符命名规则。
  • 属性类型: 扩展属性的数据类型.
  • get(): getter 方法，用于获取扩展属性的值。 必须提供 getter 方法。 在 getter 方法中，可以使用 this 关键字访问接收者类型的实例。
  • set(value): setter 方法 (可选，只在 var 类型的扩展属性中需要)，用于设置扩展属性的值。 setter 方法有一个默认的参数 value，表示要设置的新值。

例子：为 String 类添加一个扩展属性，获取字符串的首字母。

另一个例子：为 List<Int> 类添加一个扩展属性，计算列表中所有正数的和。

扩展属性不能有 backing field: 扩展属性本身不能存储数据，它只是提供了一种访问属性的语法，实际的属性值需要通过 getter (和 setter) 方法计算或从其他地方获取。 如果你需要添加真正存储数据的属性，你需要修改原始类。

扩展函数的可见性范围

扩展函数和扩展属性的可见性范围由其声明位置决定，遵循 Kotlin 的可见性修饰符 (public, internal, private, protected)。

  • 顶层扩展函数/属性 (Top-level extensions): 声明在文件顶层的扩展函数和属性，默认是 public 可见的，可以在任何地方访问。 可以使用 internal 或 private 修饰符限制其可见性。
  • 类成员扩展函数/属性 (Member extensions): 在一个类内部声明的扩展函数和属性，其可见性取决于外部类的可见性，并且只能在外部类的实例上调用。 成员扩展函数可以访问外部类的 private 和 protected 成员。 成员扩展函数不常用，一般更推荐使用顶层扩展函数。

10.5 使用场景和优势

  • 为现有类添加实用工具函数: 例如为 String 添加 isEmail(), reverse(), 为 List 添加 sumOfPositiveNumbers(), removeIf() 等。
  • 提高代码可读性和组织性: 将与特定类相关的扩展函数放在一起，可以更好地组织代码，提高可读性。 例如，可以将与字符串操作相关的扩展函数放在一个文件中，与集合操作相关的扩展函数放在另一个文件中。
  • 领域特定语言 (DSL): 扩展函数是构建 DSL 的重要工具。 通过扩展函数，可以创建更流畅、更自然的 API。 例如 Kotlin 的 Anko 库 (Android 开发) 和 Exposed 库 (数据库访问) 都大量使用了扩展函数来构建 DSL。

总结: 扩展函数和扩展属性是 Kotlin 中非常实用的特性，它们提供了一种优雅的方式来扩展现有类的功能，而无需修改源代码或使用继承。 合理使用扩展函数和扩展属性可以使代码更简洁、更易读、更易维护。
