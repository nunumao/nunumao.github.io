第九章： Maven 插件

9.1 什么是 Maven 插件？

Maven 插件 (Plugin) 是 Maven 功能的 扩展单元。 它是一组预定义的代码，用于执行特定的构建任务。 可以将 Maven 插件看作是 Maven 的 工具箱，每个插件都是一个工具，可以完成特定的任务。

Maven 插件的主要作用：

  • 扩展 Maven 功能： Maven 核心框架只提供基本的构建流程和生命周期，具体的构建任务 (例如编译、测试、打包、部署等) 都是通过插件来实现的。 Maven 插件可以扩展 Maven 的功能，使其能够完成各种复杂的构建任务。
  • 定制构建过程： 通过配置和使用不同的插件，可以定制 Maven 的构建过程，满足不同项目的构建需求。
  • 代码重用： Maven 插件是可重用的组件，可以在不同的 Maven 项目中共享和使用。
  • 社区生态丰富： Maven 社区提供了大量的插件，涵盖了各种常见的构建任务和工具集成 (例如代码检查、代码格式化、代码覆盖率、静态分析、代码混淆、版本控制集成、持续集成集成、部署到各种应用服务器或云平台等)。

9.2 Maven 插件的分类

Maven 插件可以分为两种类型：

  1. 构建插件 (Build Plugin)
  2. 报告插件 (Reporting Plugin)

9.2.1 构建插件 (Build Plugin)

  • 作用： 在构建过程中执行各种构建任务。 构建插件与 Maven 构建生命周期绑定，在生命周期的不同阶段执行不同的构建任务。
  • 配置位置： 构建插件配置在 pom.xml 文件的 <build> -> <plugins> 元素中。
  • 常用构建插件：
    • maven-compiler-plugin: 用于编译源代码和测试代码。
    • maven-surefire-plugin: 用于运行单元测试。
    • maven-failsafe-plugin: 用于运行集成测试。
    • maven-jar-plugin: 用于打包 JAR 文件。
    • maven-war-plugin: 用于打包 WAR 文件 (Web 应用)。
    • maven-ear-plugin: 用于打包 EAR 文件 (企业级应用)。
    • maven-resources-plugin: 用于处理资源文件。
    • maven-install-plugin: 用于将 package 安装到本地仓库。
    • maven-deploy-plugin: 用于将 package 部署到远程仓库。
    • maven-clean-plugin: 用于清理项目。
    • maven-assembly-plugin: 用于自定义构建输出文件的结构和内容 (例如创建 ZIP 包、TAR 包、自定义目录结构)。
    • maven-dependency-plugin: 用于依赖管理相关任务 (例如查看依赖树、分析依赖关系、拷贝依赖)。
    • maven-antrun-plugin: 用于在 Maven 构建过程中执行 Ant 任务。
    • exec-maven-plugin: 用于在 Maven 构建过程中执行系统命令或 Java 程序。

9.2.2 报告插件 (Reporting Plugin)

  • 作用： 生成项目报告。 报告插件用于生成各种项目报告，例如项目站点文档、代码质量报告、测试覆盖率报告、依赖分析报告等。
  • 配置位置： 报告插件配置在 pom.xml 文件的 <reporting> -> <plugins> 元素中。 报告插件通常与 site 生命周期配合使用，在 mvn site 命令执行时生成报告。
  • 常用报告插件：
    • maven-site-plugin: 用于生成项目站点文档。
    • maven-project-info-reports-plugin: 用于生成项目信息报告 (例如项目依赖报告、项目插件报告、项目团队报告等)。
    • maven-javadoc-plugin: 用于生成 Javadoc 文档。
    • maven-jxr-plugin: 用于生成带有代码 cross-reference 的 HTML 报告。
    • maven-surefire-report-plugin: 用于生成单元测试报告。
    • maven-checkstyle-plugin: 用于生成 Checkstyle 代码风格检查报告。
    • maven-pmd-plugin: 用于生成 PMD 代码缺陷检查报告。
    • maven-dependency-analyzer-plugin: 用于生成依赖分析报告。
    • jacoco-maven-plugin: 用于生成 JaCoCo 代码覆盖率报告。

9.3 插件的坐标 (Plugin Coordinates)

Maven 插件也使用 Maven 坐标 (GroupId, ArtifactId, Version) 进行唯一标识。 插件的坐标与依赖库的坐标格式相同。

  • groupId: 插件的组织 ID，通常以 org.apache.maven.plugins 开头 (官方插件) 或以其他组织或公司的域名反写开头 (第三方插件)。
  • artifactId: 插件的名称，例如 maven-compiler-plugin, maven-surefire-plugin, jacoco-maven-plugin。
  • version: 插件的版本号，例如 3.8.1, 2.22.2, 0.8.7。

9.4 插件的使用

Maven 插件的使用主要包括以下几个方面：

  1. 声明插件依赖 (可选)
  2. 配置插件
  3. 执行插件目标

9.4.1 声明插件依赖 (可选)

对于 构建插件，通常 不需要显式声明插件依赖。 Maven 会根据插件的坐标自动下载和使用插件。 Maven 核心插件 (groupId 为 org.apache.maven.plugins) 默认包含在 Maven 发行版中，可以直接使用，无需声明依赖。 第三方插件 (groupId 不是 org.apache.maven.plugins) 需要在 pom.xml 文件中声明插件依赖，Maven 才能找到和使用插件。

对于 报告插件，需要在 pom.xml 文件的 <reporting> -> <plugins> 元素中声明插件依赖，Maven 才能在生成站点报告时使用报告插件。

9.4.2 配置插件

Maven 插件的配置可以在 pom.xml 文件中进行。

  • 构建插件配置: 在 pom.xml 文件的 <build> -> <plugins> 元素中配置构建插件。

  • 报告插件配置: 在 pom.xml 文件的 <reporting> -> <plugins> 元素中配置报告插件。

  • <configuration> 元素: 用于配置插件的参数。 不同的插件有不同的参数，插件的参数可以在插件的官方文档中查阅。 插件参数通常以 XML 元素的形式配置在 <configuration> 元素中。

  • 插件版本: 建议 显式指定插件的版本号，避免使用默认版本，确保构建的稳定性和可重复性。 插件版本可以在 <plugin> 元素的 <version> 子元素中指定。

9.4.3 执行插件目标

Maven 插件的目标 (Goal) 可以通过以下几种方式执行：

  1. 绑定到生命周期阶段： Maven 插件的目标可以绑定到 Maven 生命周期阶段，当 Maven 执行到该生命周期阶段时，会自动执行与之绑定的插件目标。 Maven 已经为一些常用的生命周期阶段默认绑定了插件目标 (例如 compile 阶段绑定了 maven-compiler-plugin:compile 目标，test 阶段绑定了 maven-surefire-plugin:test 目标，package 阶段绑定了 maven-jar-plugin:jar 目标)。 也可以在 pom.xml 文件中自定义插件目标与生命周期阶段的绑定关系 (使用 <executions> 元素)。

9.5 插件的生命周期绑定 (Plugin Bindings)

Maven 为每个生命周期阶段都 预定义 了一组 默认的插件绑定 (Default Plugin Bindings)。 这些默认绑定定义了在每个生命周期阶段默认执行的插件目标。 Maven 核心插件通常都绑定到了 default 生命周期的不同阶段。

可以使用 mvn help:describe -Dplugin=<plugin-artifactId> -Dgoal=<goal> 命令查看插件目标的详细信息，包括插件目标绑定的生命周期阶段。 例如，查看 maven-compiler-plugin:compile 目标的绑定信息：

9.6 插件的管理 (pluginManagement)

与 dependencyManagement 类似，Maven 也提供了 pluginManagement 元素，用于 统一管理插件的版本和配置。 pluginManagement 元素定义在 <build> 元素下，与 <plugins> 元素平级。

  • pluginManagement 的作用：
    • 统一插件版本管理： 在 pluginManagement 中定义插件的版本号，在 <plugins> 元素中使用插件时可以省略版本号，版本号从 pluginManagement 中继承。
    • 统一插件配置管理： 在 pluginManagement 中定义插件的通用配置，在 <plugins> 元素中使用插件时可以继承 pluginManagement 中的配置。
    • 不实际执行插件： pluginManagement 中定义的插件配置 不会被实际执行，只是用来管理插件版本和配置。 需要在 <plugins> 元素中显式声明插件，才能实际使用插件。
