第六章： 构建 Profile

6.1 什么是 Maven Profile？

Maven Profile 用于 自定义不同环境或构建目标的构建过程。 它们允许你为以下情况定义不同的配置：

  • 不同环境： 开发环境、测试环境、预发布环境、生产环境等等。
  • 不同操作系统： Windows, Linux, macOS。
  • 不同 JDK 版本。
  • 不同构建目标： 构建 JAR 包、WAR 包，或者运行集成测试等。

为什么要使用 Profile？

  • 环境特定的配置： 你可以为每个环境定义不同的配置，例如数据库连接信息、日志级别、资源文件等等。
  • 构建自定义： 你可以根据不同的条件自定义构建过程，例如激活不同的插件或插件版本。
  • 避免直接修改 POM 文件： Profile 允许你在不修改主 POM 文件的情况下切换配置，保持 POM 文件的整洁和可维护性。
  • 灵活性和可重用性： Profile 提供了一种灵活和可重用的方式来管理不同的构建配置。

6.2 在 pom.xml 中定义 Profile

Profile 是在你的 pom.xml 文件中的 <profiles> 元素内定义的。 每个 profile 由一个 <profile> 元素定义，并且拥有一个唯一的 <id>。

Profile 的基本结构：

<profile> 元素的主要子元素：

  • <id> (必需): Profile 的 ID，用于在 Maven 中唯一标识一个 Profile。 ID 通常使用简短且具有描述性的名称，例如 dev, test, prod, windows, jdk1.8 等。

  • <activation> (可选): Profile 激活配置，用于定义在什么条件下激活该 Profile。 如果没有 <activation> 元素，则 Profile 默认不会自动激活，需要显式激活。 <activation> 元素可以包含以下子元素来定义激活条件：

    • <activeByDefault>: 设置为 true 时，该 Profile 将 默认激活。 一个 POM 文件中只能有一个 Profile 设置为 activeByDefault 为 true。
    • <jdk>: 根据 JDK 版本 激活 Profile。 例如 <jdk>1.8</jdk> 表示当 JDK 版本为 1.8 时激活 Profile，<jdk>[1.7,)</jdk> 表示当 JDK 版本大于等于 1.7 时激活 Profile。
    • <os>: 根据 操作系统 激活 Profile。 可以根据操作系统的 family (例如 windows, unix, mac), name, arch, version 等属性激活 Profile。
    • <property>: 根据 Maven 属性 或 系统属性 激活 Profile。 例如 <property><name>env</name><value>dev</value></property> 表示当 Maven 属性或系统属性 env 的值为 dev 时激活 Profile。
    • <file>: 根据 文件存在性 激活 Profile。 可以根据文件是否存在 (<exists>) 或文件不存在 (<missing>) 激活 Profile。

  • 配置元素 (可选): Profile 中可以配置以下元素，用于 覆盖或修改 POM 文件中原有的配置：

    • <properties>: 定义 Profile 特有的 Maven 属性。 这些属性可以在 POM 文件和其他配置文件中引用。
    • <dependencies>: 依赖管理，可以在 Profile 中添加、移除或替换依赖。
    • <repositories>: 仓库配置，可以在 Profile 中添加、移除或替换仓库。
    • <pluginRepositories>: 插件仓库配置，可以在 Profile 中添加、移除或替换插件仓库。
    • <build>: 构建配置，可以修改构建相关的配置，例如：
      • <defaultGoal>: 定义默认的 Maven 目标。
      • <resources>: 资源文件配置。
      • <testResources>: 测试资源文件配置。
      • <plugins>: 插件配置。
      • <filters>: 过滤器配置。
      • <extensions>: 扩展配置。
      • <pluginManagement>: 插件管理配置。

6.3 Profile 激活方式

Maven 提供了多种 Profile 激活方式：

  1. 默认激活 (<activeByDefault>)

    在 Profile 的 <activation> 元素中设置 <activeByDefault>true</activeByDefault> 可以将该 Profile 设置为 默认激活。 在一个 POM 文件中，只能有一个 Profile 设置为默认激活。

    当没有显式指定激活 Profile 时，默认激活的 Profile 会被使用。

  2. 命令行激活 (-P <profile-id>)

    在执行 Maven 命令时，可以使用 -P <profile-id> 参数来 显式激活指定的 Profile。 可以同时激活多个 Profile，使用逗号分隔 Profile ID，例如 -P dev,debug。

  3. settings.xml 文件激活 (<activeProfiles>)

    可以在 Maven 的 settings.xml 文件中配置 <activeProfiles> 元素来 全局激活 Profile。 在 settings.xml 中激活的 Profile 会 覆盖 POM 文件中默认激活的 Profile。

  4. 自动激活 (基于 <activation> 元素)

    通过配置 <activation> 元素中的 <jdk>, <os>, <property>, <file> 等子元素，可以实现 自动激活 Profile。 当满足 <activation> 元素定义的条件时，Profile 会自动被激活。

6.5 Profile 的优先级

当多个 Profile 同时被激活时，Maven 会按照一定的优先级规则合并 Profile 的配置。 Profile 的优先级顺序 (从高到低)：

  1. 命令行激活的 Profile (-P)
  2. settings.xml 文件中 activeProfiles 激活的 Profile
  3. 用户 settings.xml 文件中 profiles 定义的 Profile
  4. 全局 settings.xml 文件中 profiles 定义的 Profile
  5. POM 文件中 activeByDefault 激活的 Profile
  6. POM 文件中 profiles 定义的 Profile (按定义顺序)

优先级高的 Profile 配置会覆盖优先级低的 Profile 配置。 如果多个 Profile 配置了相同的元素，则优先级最高的 Profile 配置生效。
