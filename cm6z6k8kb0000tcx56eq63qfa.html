Kotlin (七) | 集合

KOTLIN 2025-02-10 15 分钟阅读 0

集合 (Collections)

集合是编程中非常常用的数据结构，用于存储和操作一组对象。 Kotlin 提供了丰富的集合类型，并且对集合操作提供了强大的支持。 Kotlin 的集合框架设计简洁、安全且高效。

List (列表), Set (集合), Map (映射)

Kotlin 集合框架主要包含三种基本类型的集合：

  • List (列表): 有序集合，元素可以重复。 List 中的元素按照插入顺序排列，可以通过索引访问元素。 例如，存储一组学生的名字列表，允许有同名的学生。

    • 特点:
      • 有序 (Ordered): 元素按照插入顺序存储。
      • 允许重复元素 (Duplicates Allowed): 可以包含相同的元素。
      • 可以通过索引访问元素 (Indexed Access): 可以使用索引 (从 0 开始) 获取特定位置的元素。

  • Set (集合): 无序集合，元素不允许重复。 Set 主要用于存储唯一的元素，例如存储一组学生的学号，学号应该是唯一的。

    • 特点:
      • 无序 (Unordered): 元素没有特定的顺序 (通常基于哈希值存储，顺序不保证稳定)。
      • 不允许重复元素 (No Duplicates): 每个元素在 Set 中只能出现一次。
      • 高效的元素查找 (Efficient Element Lookup): Set 提供了快速的元素查找操作。

  • Map (映射): 键值对 (key-value pairs) 的集合。 每个元素都是一个键值对，键 (key) 是唯一的，值 (value) 可以重复。 Map 用于存储键值对关系的数据，例如存储学生的学号和姓名，学号是键，姓名是值。

    • 特点:
      • 键值对存储 (Key-Value Pairs): 存储键和值之间的关联关系。
      • 键唯一 (Unique Keys): 每个键在 Map 中只能出现一次。
      • 通过键访问值 (Value Access by Key): 可以使用键快速查找对应的值。

可变集合和不可变集合 (Mutable and Immutable Collections)

Kotlin 区分可变集合 (Mutable Collections) 和不可变集合 (Immutable Collections)。

  • 不可变集合 (Immutable Collections): 创建后内容不能被修改 (不能添加、删除或修改元素)。 不可变集合是线程安全的，更易于理解和维护，推荐尽可能使用不可变集合。

    • 创建不可变集合的函数:
      • listOf(...): 创建不可变的 List
      • setOf(...): 创建不可变的 Set
      • mapOf(...): 创建不可变的 Map

  • 可变集合 (Mutable Collections): 创建后内容可以被修改 (可以添加、删除、修改元素)。 可变集合提供了修改集合内容的方法。

    • 创建可变集合的函数:
      • mutableListOf(...): 创建可变的 List
      • mutableSetOf(...): 创建可变的 Set
      • mutableMapOf(...): 创建可变的 Map

选择可变性: 优先使用不可变集合，除非你需要修改集合的内容。 不可变集合可以提高程序的安全性和可预测性。 如果需要修改集合，才使用可变集合。

类型推断和集合: Kotlin 能够推断集合的元素类型。 也可以显式指定元素类型，例如 List<String>, Set<Int>, Map<String, Double>。

8.3 集合的常用操作

Kotlin 集合框架提供了丰富的操作函数，用于对集合进行各种操作，例如添加、删除、查找、遍历、过滤、转换等。 这些操作函数通常以扩展函数的形式提供，可以直接在集合对象上调用。

常用操作函数 (部分):

  • 添加元素:

    • add(element) (MutableList, MutableSet): 添加单个元素
    • addAll(collection) (MutableList, MutableSet): 添加另一个集合的所有元素
    • put(key, value) (MutableMap): 添加键值对
    • putAll(map) (MutableMap): 添加另一个 Map 的所有键值对

  • 删除元素:

    • remove(element) (MutableList, MutableSet): 删除指定元素
    • removeAll(collection) (MutableList, MutableSet): 删除与另一个集合中元素相同的元素
    • clear() (MutableList, MutableSet, MutableMap): 清空集合
    • remove(key) (MutableMap): 删除指定键的键值对

  • 查找元素:

    • contains(element) 或 containsAll(collection) (List, Set): 检查集合是否包含指定元素或所有元素
    • containsKey(key) 或 containsValue(value) (Map): 检查 Map 是否包含指定键或值
    • get(index) (List): 获取指定索引位置的元素
    • get(key) 或 getOrDefault(key, defaultValue) (Map): 获取指定键的值

  • 遍历集合:

    • for (element in collection): 迭代集合中的每个元素
    • forEach { element -> ... }: 对集合中的每个元素执行操作 (Lambda 表达式)
    • iterator(): 获取迭代器，手动遍历集合

  • 过滤集合:

    • filter { element -> condition }: 过滤满足条件的元素，返回新的 List
    • filterNot { element -> condition }: 过滤不满足条件的元素，返回新的 List
    • filterNotNull() (List<T?>): 过滤掉 List 中的 null 元素，返回新的 List

  • 转换集合:

    • map { element -> transform }: 将集合中的每个元素转换为另一个值，返回新的 List
    • flatMap { element -> collection }: 将集合中的每个元素转换为一个集合，并将所有子集合合并成一个 List
    • toSet(), toList(), toMap(): 将集合转换为 Set, List, Map

  • 排序集合:

    • sorted(), sortedDescending() (List, Set): 对集合进行排序，返回新的 List
    • sortedBy { element -> key }, sortedByDescending { element -> key } (List, Set): 根据指定的 key 对集合进行排序，返回新的 List

  • 聚合操作:

    • sum(), average(), count(), max(), min() (List, Set): 计算集合元素的总和、平均值、数量、最大值、最小值
    • reduce { accumulator, element -> operation }: 将集合元素聚合为一个值 (例如求和、求积)
    • fold(initialValue) { accumulator, element -> operation }: 与 reduce 类似，但可以指定初始值

集合操作链 (Chaining Collection Operations):

Kotlin 集合操作函数可以链式调用，将多个操作组合在一起，以简洁、流畅的方式完成复杂的数据处理。
