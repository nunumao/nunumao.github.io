接口 (Interfaces)

接口是面向对象编程中另一个重要的概念。 接口定义了一组方法签名，但不提供方法的具体实现。 类可以实现一个或多个接口，从而承诺提供接口中定义的方法的具体实现。 接口主要用于定义契约 (contract)，实现多态 (polymorphism) 和实现行为的 “多重继承”。

接口的定义

在 Kotlin 中，我们使用 interface 关键字来定义接口。

语法:

  • interface: 关键字，表示这是一个接口定义。
  • 接口名: 接口的名称，遵循标识符命名规则，通常使用 PascalCase 命名法 (例如 Clickable, Drawable, Comparable)。
  • 接口体: 用花括号 {} 包裹的部分，包含接口的成员 (属性声明、方法声明)。

接口成员:

  • 抽象属性 (Abstract Properties): 接口可以声明属性，但不能提供初始值。 实现接口的类必须提供属性的实现 (必须是 override 并且需要提供初始值或通过 getter/setter 初始化)。 从 Kotlin 1.4 开始，接口也可以声明带有 backing field 的属性，从而在接口中提供属性的默认实现，但这比较少见，我们先关注抽象属性。

  • 抽象方法 (Abstract Methods): 接口中声明的方法默认是抽象的 (abstract)，不需要显式使用 abstract 关键字。 抽象方法没有方法体，只有方法签名。 实现接口的类必须提供抽象方法的具体实现 (必须使用 override 关键字)。

  • 默认方法 (Default Methods): 从 Java 8 开始，接口可以包含默认方法，Kotlin 也支持。 默认方法在接口中提供方法的默认实现。 实现接口的类可以选择使用默认实现，也可以重写默认方法提供自己的实现。 默认方法使用 fun 关键字并提供方法体。

类实现接口

接口的多重继承 (Multiple Inheritance)

与类只能单继承不同，一个类可以实现多个接口，这被称为接口的多重继承。 接口的多重继承允许一个类拥有多个接口定义的行为，从而实现更灵活的组合和复用。

例子见 7.2 中的 Car 类，它同时实现了 Drivable 和 قابلRepair 两个接口，拥有了这两个接口定义的行为。

接口的默认方法 (Default Methods in Interfaces)

接口的默认方法提供了一种在不破坏已实现接口的类的情况下，向接口添加新方法的机制。 实现接口的类可以选择使用默认实现，也可以重写默认方法提供自己的实现。

接口继承接口: 接口也可以继承其他接口，使用冒号 : 语法。 一个接口可以继承多个接口。 继承接口的接口会继承父接口的所有成员。

接口与抽象类的对比 (总结):

特性 接口 (Interface) 抽象类 (Abstract Class)
关键字 interface abstract class
继承/实现 类可以实现多个接口 (多重继承行为) 类只能继承一个抽象类 (单继承)
构造函数 没有构造函数 可以有构造函数
属性 可以声明抽象属性 (需要实现类提供实现)，Kotlin 1.4+ 可以声明带有 backing field 的属性 (提供默认实现) 可以声明抽象属性和非抽象属性 (可以有状态)
方法 可以声明抽象方法和默认方法 可以声明抽象方法和非抽象方法 (可以提供部分实现)
实例化 不能被实例化 不能被实例化
主要用途 定义契约，实现多态，实现行为的 “多重继承” 定义类的通用模板，代码复用，部分实现，继承层次的骨架
应用场景 定义组件之间的协议，插件式架构，实现各种行为的组合 定义一组相关类的共同特性，提取公共代码，构建框架
